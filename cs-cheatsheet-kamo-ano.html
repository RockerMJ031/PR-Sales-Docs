Claude Code + Lark — Customer Support Cheat Sheet

For Kamo & Ano · setup ~30 min · full guide in the Training Library doc

1 Install Claude Code

Mac
curl -fsSL https://claude.ai/install.sh | bash
Windows (PowerShell — prompt shows PS C:\)
irm https://claude.ai/install.ps1 | iex
Windows: also install Git for Windows (git-scm.com). Then run claude & log in via browser.
✅ Verify in a fresh terminal: claude --version → version number = success. Health check: claude doctor.
💳 Needs a paid Claude plan — message MJ first, just create the claude.ai account with your work email.

2 Connect Lark

Mac
brew install node
npx @larksuite/cli@latest install
Windows: install Node.js LTS (nodejs.org), then npx @larksuite/cli@latest install in a fresh PowerShell.
Configure & authorise (minimal recommended permissions — grant more later only when a task asks):
lark-cli config init
lark-cli auth login --recommend
lark-cli auth status
✅ lark-cli auth status shows you logged in + scopes = setup done.
Then tell Claude: "Read the CS Helper playbook doc (link in card 8) and keep it as your ticket reference."

3 Daily habits

🧹/clear after every task. One task = one conversation. Long chats get slow.
🎯Say the outcome, not the steps. "Draft a warm reply confirming removal, with effective date" — that's a good ask.
🔍Verify → show → act. Make Claude check live data and show you before anything is sent or changed.
🧠"Remember this." Corrections go to memory — it stops repeating the mistake.

⭐ The Golden Rule

Verify-and-confirm, never silent-write. Claude reads freely — but never deactivates a student, creates a ticket, or sends an email without your explicit yes. Passwords masked by default. Safeguarding / Compliance / Legal → escalate to Bella/DSL, never auto-resolve.
